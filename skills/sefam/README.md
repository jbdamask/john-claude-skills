# SEFAM — Simple Enough For A Manager

![SEFAM](./SEFAM.png)

SEFAM takes a technical subject and produces a short, plain-language write-up that a smart but **non-technical** reader — like a manager — can fully understand. No jargon, but never dumbed down.

It works on either:

- **A change** — a PR, diff, commit, bug fix, or issue.
- **A concept** — a function, process, architecture, system, service, or design decision.

The skill reads the actual source first (the diff, the code, the docs), translates every technical term into what it means for people, money, or the product, and leads with impact or purpose rather than implementation. It sticks to facts it can trace to the source — no invented numbers, costs, or timeframes — and always includes an honest caveat instead of overselling.

## Output

The deliverable is a **great-looking, self-contained web page** (an Artifact). The skill auto-picks a fitting visual style from a catalog of eleven — Academic Scholar, Brutalist, Neo-brutalist, De Stijl, Earth Tones, Kids Starface, Tactile Maximalism, Vintage Travel Poster, Miyazaki Sketchbook, Glassmorphism, and mellowkenneth-sketch — tells you which it chose and why, and re-renders in a different style on request.

## How to use

Point the skill at what you want explained and say any of:

- "sefam" / "SEFAM"
- "explain this simply" / "simple enough for a manager"
- "explain this to a non-technical person"
- "manager-friendly summary" / "plain-English explanation"
- "de-jargon this"

See [`SKILL.md`](./SKILL.md) for the full method.
